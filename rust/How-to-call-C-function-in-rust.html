    How to call RUST function from C

    I make programs mainly using C or C++ language. Recently I‚Äôm studying Rust. So I wanna interact to C and Rust. I want to link each source file(.c or .rs) after compiling it as an object file.

    Let‚Äôs say I want to call a Rust function in C.
    How do I do this? What does it mean to call a rust function from c? Actually I don‚Äôt know about it. So I Searching for google. So many keyword pop out!! What caught my eye was the FFI(Foreign Function Interface).

    Let‚Äôs implement C function for use in Rust. Plz follow this code.

    // file : main.c
#include <stdio.h>

void c_print_fn(const char* str) {
  printf("%s\n", str);
}

int main() {
  printf("Hello World\n");

  // this is test for c
  c_print_fn("Calling in C");

  return 0;
}

    When We build and run the program, We‚Äôll see something like this:

    $ clang -o main main.c
$ ./main
Hello World
Calling in C

    Great! Next step is make Rust source file.
    First, let‚Äôs make the main function.

    // file : rust_ffi.rs
fn call_from_c(num: i32) -> i32 {
  num
}

// Just function test... later delete
fn main() {
  println!("{}", call_from_c(10));
}

    We will use only rustc is the compiler for the Rust programming language, provided by the project itself.

    $ rustc rust_ffi.rs
$ ./rust_ffi

    Now step is calling RUST function from C.
    Write the function to use for c in pub extern "C" fn ~~~.
    At this time, it should be the same as the prototype implemented in C.

    // file : rust_ffi.rs
pub extern "C" fn call_from_c(num: i32) -> i32 {
  num
}

// TODO : later remove main function
fn main() {
  println!("{}", rust_pow_num(10));
}

    Okay‚Ä¶ Let‚Äôs call the rust function from C

    // file : main.c

  printf("%s\n", str);
}

int main() {
  printf("Hello World\n");
  
  // this is test for c
  c_print_fn("Calling in C");

  // rust fn call!!
  printf("%d\n", call_from_c(211));

  return 0;

    Compiler doesn‚Äôt know where call_from_c function is. So declare function and parameter same as rust function.
    fix like this :

    // file : main.c

  printf("%s\n", str);

  // this is test for c
  c_print_fn("Calling in C");

  // rust fn call!!
  printf("%d\n", call_from_c(211));

    Hum‚Ä¶ Why don‚Äôt make executable file?‚Ä¶ Let‚Äôs fix it.
    I think rust file(rust_ffi.rs) has a problem‚Ä¶ because multiple definition of ‚Äòmain‚Äô is very important build error.
    rust_ffi.rs file has a main function and comment.
    Delete main as comment. like this:

    // file : rust_ffi.rs
pub extern "C" fn call_from_c(num: i32) -> i32 {
  num
